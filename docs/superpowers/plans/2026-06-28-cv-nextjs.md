# CV Next.js Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the default `create-next-app` boilerplate with a single static CV page that visually replicates the Canva-derived HTML the user provided, using placeholder content.

**Architecture:** One Next.js App Router page (`app/page.js`) composed of two presentational components — a dark sidebar (`CvSidebar`) and a white main content area (`CvMain`) — laid out side-by-side inside an A4-proportioned container. No state, no data fetching, no routes beyond `/`.

**Tech Stack:** Next.js 16.2.9 (App Router), React 19.2.4, JavaScript (no TypeScript), Tailwind CSS v4 (already installed natively via `@tailwindcss/postcss`), `next/font/google` for DM Sans.

## Global Constraints

- No TypeScript — all files are `.js` / `.jsx`-in-`.js` (project was scaffolded with `--js`).
- No new npm dependencies (no icon libraries, no `next/image` remote-domain config) — matches spec's "Fuera de alcance".
- Placeholder content from the source HTML (name, contact, skills, experience, education) is kept verbatim — it is not the user's real data, the user will edit it later.
- Profile photo stays as the original Pexels stock URL, loaded via a plain `<img>`, not `next/image` (avoids editing `next.config.mjs` for remote image patterns, which is out of scope).
- Colors in the source HTML are inline RGB values that are an exact match for Tailwind's default `slate` palette (verified: `rgb(30,41,59)`=slate-800, `rgb(15,23,42)`=slate-900, `rgb(71,85,105)`=slate-600, `rgb(51,65,85)`=slate-700, `rgb(100,116,139)`=slate-500, `rgb(148,163,184)`=slate-400, `rgb(203,213,225)`=slate-300) — use the Tailwind class names, not inline `style` props.
- Exact font sizes from the source HTML (11px, 12px, 13px, 15px, 16px, 26px) are preserved via Tailwind arbitrary-value classes (e.g. `text-[15px]`) since they don't map cleanly to Tailwind's default type scale.
- `npm run lint` is expected to print one `warn` for `@next/next/no-img-element` on the profile photo — this is intentional (see photo constraint above) and is not a failure; do not "fix" it by switching to `next/image`.
